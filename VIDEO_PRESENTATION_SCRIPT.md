# SEMS Video Presentation Script

Use this script as a guide when recording your project demonstration. It highlights all the critical features, UI enhancements, and design patterns to ensure you get full marks!

---

## 🟢 Part 1: Introduction (0:00 - 0:30)
**Action:** Start with the Application open on the Login Screen.
**Script:**
> "Hello, this is [Your Name] from Group [X]. Today, I will be presenting our Software Design & Analysis project: The Sports Equipment Management System (SEMS). This JavaFX application was built using Maven, and heavily utilizes Object-Oriented principles and Design Patterns like Singleton, Observer, and Factory to provide a robust role-based system."

## 🔵 Part 2: The Student Workflow (0:30 - 1:30)
**Action:** Type `ali@sems.edu` and `Student1!`. Click Login.
**Script:** 
> "First, let's log in as a Student. Our system uses a centralized `SessionManager` Singleton to handle authentication. Once logged in, the UI dynamically restricts our view based on our role."
**Action:** Click 'Browse Equipment' and click 'Borrow' on the Football. Select a date and click submit.
**Script:**
> "Here, a student can browse the catalog. I'll request to borrow a Football. Notice the UI confirms the request. Behind the scenes, an `Observer` pattern is triggered: the `InventoryManager` broadcasts an event, and the `NotificationObserver` immediately generates a notification for the student and the admin."
**Action:** Show the bell icon with the new notification, and show the 'My Active Borrows' stat card.

## 🔴 Part 3: The Admin (Sports Head) Workflow (1:30 - 2:30)
**Action:** Click Logout. Type `admin@sems.edu` and `Admin123!`. Click Login.
**Script:**
> "Now, let's log out and log in as the Sports Head. Because of our Factory pattern, the `User` object created for the admin has elevated privileges."
**Action:** Point to the Dashboard. Highlight the 'Pending Requests' stat card and the 'Pending Borrow Requests' section.
**Script:**
> "On the Admin dashboard, we can see system-wide statistics. Notice the Pending Request we just submitted is clearly visible here with an Approve and Reject button. The admin also sees a notification in their feed generated by our Observer pattern."
**Action:** Click 'Approve' on the request.
**Script:**
> "I will approve this request. The system immediately updates the status to APPROVED."

## 🟠 Part 4: Returning Equipment (2:30 - 3:00)
**Action:** Logout, log back in as the Student (`ali@sems.edu`).
**Script:**
> "Finally, let's log back in as the student to return the item."
**Action:** Go to the 'Return Equipment' screen. Select the item, choose 'Minor Repair' for the condition, and submit.
**Script:**
> "The student can return the equipment and must specify a condition report. By selecting 'Minor Repair', the system intelligently updates the equipment's global status so it can no longer be borrowed until it is fixed by the Admin."

## 🏁 Part 5: Conclusion (3:00 - 3:30)
**Action:** Show the codebase briefly if required, pointing to `DatabaseManager.java` or `Observer` package.
**Script:**
> "Throughout this project, we maintained a clean Model-View-Controller separation. By using Data Models, Business Logic Services, and UI Screens, we ensured our code is highly modular, maintainable, and strictly adheres to SOLID principles. Thank you for watching!"
